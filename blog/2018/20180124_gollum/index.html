個人用wiki(Gollum) + reveal.jsによる情報一括管理

環境

  • Ubuntu 16.04LTS

概要

  • 個人用wikiについて

  • gollum + reveal.js環境

    • gollumのセットアップ
    • gollumのカスタマイズ
    • reveal.js (+pandoc)によるスライド作成

なぜ個人用wikiをなぜ使うのか

そもそも何をしたいのか

  • ドキュメント，メモの管理
  • pdfの管理
  • png, jpegの管理
  • 数式をLaTeXで書きたい
  • 表も簡単に書きたい
    • 計算してない表Excelを無くすため
  • 階層構造を持ちたい
    • 何処に何があるかをわかりやすくするため
  • 検索機能

この辺の情報を(軽量に)一括で管理したい

よく見るもの

よくあるのはOffice製品によるドキュメント，メモ書き残し． Office製品は，蓄積した情報に対しての検索性が悪すぎるし， 開くまで時間かかるのが厳しい．重い． あと何処に何のファイルがわからなくなることが多い． 紙は検索ができないから嫌．

因みに外部サービスは？(Evernote等)

そのサービス終了すると悲しい目に合うので， データとして情報を残すものでは個人的にはあまり外部サービスは使わない主義． （このBlogも外部サービスに頼らなかったのはそういう理由がある） あとで情報を移植する可能性があるものは，移植しやすい形にして保存しておかないと面倒すぎて死んじゃう． あとかなり個人的な話だけど，Vimでゴリゴリ書きたいなぁってなるとwiki立てる方が早いというのもある．

色々探した結果Gollumというwikiにたどり着いた

Gollumの特徴

  • かなりシンプルなwikiでGit baseのwiki
  • Gitができる事前提なので，非エンジニア用wikiにはなりにくい
    • この記事では一人運用前提で話します
  • Gitリポジトリでファイルを管理する
  • Git commitで変更が反映される仕様
  • wikiのページ = 1ファイルなので，お好きなエディタでゴリゴリ書いていける
  • Markdownで書ける
  • プラグイン的なのを入れるとMediawiki文法も書ける
  • 数式はLaTeXで書ける
  • その気になってrailsを書けば拡張可能
    • あんまり日本語記事は少ない印象，拡張する時は英語で調べないと基本でてこない
    • configあたりでrailsの関数をゴニョっと書くとなんとかなる(白目)
  • サーバー立てる以外にも，PCごとにgollumをインストールして，Gitリポジトリだけリモートに置いておくという方式も取れる
    • サーバーよく分からなくてもwikiを立てられる

install

  • https://github.com/gollum/gollum 公式Readme
  • https://github.com/gollum/gollum/wiki 公式wiki
  • Windowでも建てられるっぽいけど，バグバグしいかもしれない・・・？(よく知らない)
  • install
    • sudo apt-get gem ruby-dev
    • sudo gem install gollum
  • mediawiki文法で書きたい人は
    • sudo gem install wikicloth

wikiを立てる

  • cd ~
  • mkdir wiki
    • お好きなところに
  • git init
  • vim Home.md or vim Home.mediawiki
    • なんか書く
    • お好きなエディタでどうぞ
  • gollum –allow-uploads –show-all –mathjax –config config.rb
    • それぞれのオプションの意味はreadme参照
    • Markdownしか使わない人は–live-previewしたほうがいいかも
    • .bashrcとか.zshrcとかで適当にaliasする
  • ブラウザでlocalhostアクセスすると見れる
    • http://localhost:4567/Home
    • デフォルトポートは4567

運用

  • wikiディレクトリ下に階層を作る，以下は例
 /study
 /tech
 /hobby
 /uploads
  • ファイル = ページを作る
    • ブラウザからも作成可能だけど，階層構造の中に入ってくれないのでlocalのエディタで作る
    • 拡張子は.md
      • mediawiki拡張してある場合は.mediawiki
  • ファイル編集
    • ファイル直接編集もできるし，ブラウザ上でも編集できる
  • ファイル書いたらgit add && git commit で更新される
  • ページ内リンクについて
    • [[ページ名]]
      • 基本的にページ名被りはさせないように
    • 頭に/をつけると絶対パス
  • ファイルのアップロード
    • /uploadsにpngやらpdfやらを置く
    • git add commit “update”
    • リンクを書く
      • pdfの場合 [[名前 | /uploads/hoge.pdf]]
      • pngの場合 [[/uploads/hoge.png]]
      • mp4の場合 [[/uploads/hoge.mp4]]
  • 数式
    • $ \frac{x}{y} $
    • $$ \frac{x}{y} $$

gollumを拡張

cssとconfig ファイル

  • custom.cssをwikiディレクトリに置く
  • wiki起動時は
    • gollum –config config.rb –css –allow-uploads –show-all –mathjax
  • 自分の設定ファイル https://github.com/scepter914/gollum_config
    • config
      • mediawiki の色付き表を使えるようにした
    • css
      • 編集枠を大きく
      • ページの幅ひろく
      • サイドバー狭く
      • 行間狭く
  • 設定ファイルが反映されてない時
    • gitがmasterであることを確認する
  • css勉強すれば好きにカスタマイズができる(白目)
  • カスタマイズ
    • https://github.com/gollum/gollum/tree/master/lib/gollum/public/gollum/cssにあるのを参考にしながら適当に
    • 自分のローカルだと/var/lib/gems/2.3.0/gems/gollum-4.1.0/lib/gollum/public/gollum/cssだったのでご参考までに

PlantUML

  • モデリング言語
    • 詳しくはggってください
  • 公式 https://github.com/gollum/gollum/wiki/Custom-PlantUML-Server#gollum-configuration
  • ubuntu16.04だとppa追加が必要
  • sudo add-apt-repository ppa:openjdk-r/ppa
  • sudo apt-get update
  • sudo apt-get install openjdk-7-jdk
  • sudo apt-get install maven graphviz git-core
  • git clone https://github.com/plantuml/plantuml-server.git
  • cd plantuml-server
  • mvn jetty:run

  • http://localhost:8080/plantumlで見られるはず

  • wikiディレクトリの中に置くなら.gitignoreしておく

reveal.js (+ pandoc)システム

install

  • reveal.js
    • Markdownからスライドを作成できる神ツール
    • 詳しくはggってください
    • 公式 https://revealjs.com/#/
  • pandoc
    • 文書を別の形式に買える神ツール
    • 今回はmarkdown to revealjsをする
    • 詳しくはggってください
  • revealjs install
    • 参考 公式 https://github.com/hakimel/reveal.js/#installation
    • git clone https://github.com/hakimel/reveal.js.git
  • pandoc install
    • sudo apt-get install pandoc

環境構築

  • revealjsディレクトリをwikiの下に置く
  • .gitignoreに追加する
  • slide.shを作成する
    • ファイル場所は適宜
var=`grep -ic '# ' $1`
echo "スライド数"$var  
cp $1 ~/wiki/.reveal.js/index.md
cp -r ./pic ~/wiki/.reveal.js/
cd ~/wiki/.reveal.js/
pandoc -f markdown index.md -t revealjs -V theme:black --template=$HOME/wiki/uploads/code/pandoc_template/slide_default.html --highlight-style=espresso --slide-level=2 -o index.html
google-chrome index.html
  • .bashrc等に
    • alias slide=“$HOME/wiki/uploads/code/slide.sh”
    • を追加
